## 1. ゴール（MVP）
スマホで写真1枚 → ①カード特定 → ②素体/PSA10相場を即表示 → ③センタリング補助線（判定は人）。
用途: **店頭仕入の即断**。「これ何で、素体いくら、PSA10いくら」が3秒で分かる。

## 2. 核心アイデア：相場は既存インフラの再利用（新技術を作らない）
```
写真 → OCRで型式(例 110/080)を読む → GRのslug化(m2-110-080) → GR個別ページ
     → カード名・素体相場・PSA10相場
```
これは AgiMa が毎晩やってる相場取得と同じ仕組み。カード特定＋相場は「新規開発」ではなく「既存の延長」。

## 3. アーキテクチャ（MVP・4部品）
1. **撮影**: スマホカメラ（PWA=モバイルWebアプリ。アプリストア不要・URLで開く。getUserMedia）。型式を枠ガイドで撮らせる
2. **カード特定**: 型式領域をOCR → GR slug化（小文字・スペース/スラッシュをハイフン。SV-Pプロモは番号先頭）
3. **相場表示**: GR（算定値・確度/取得日つき）＋スニダン現物リンク（鉄則: 単一値で断定しない）
4. **センタリング補助**: canvasで外枠/内枠を検出、上下・左右の余白比を線+数値で表示。**判定せず人が見る**（肌感が主）

## 4. 技術スタック（推奨）
- **PWA（モバイルWebアプリ）**: アプリストア不要、URLで開く、カメラはWeb API。仕入入口と同じ"スマホで開くだけ"の思想
- **OCR**: 精度優先でクラウドOCR、or Tesseract.js（オフライン）。型式の数字/スラッシュ狙い
- **センタリング**: OpenCV.js（canvas画像処理）
- **相場**: §5で決定

## 5. 唯一の技術判断（設計で詰める）＝相場の取り方
GRはJS描画で、リアルタイム取得はヘッドレスブラウザが要る（重い）。
- **A（MVP推奨）**: Coworkが夜間に"広めの価格キャッシュ"を作り、アプリはそれを引く。オフライン可・速い・**店頭で電波弱くても強い**。ホスティングは静的PWA＋キャッシュをクラウド（Drive等）に
- **B**: 都度ヘッドレスfetch。最新だがバックエンド常時稼働が要る・重い

※ 店頭=LAN外なので「PC常時起動のローカルサーバ」は不可。キャッシュ方式(A)が現実的。

## 6. リスクと対処
- **OCR精度**（小文字・グレア・角度）: 枠ガイド＋明るさ要求で緩和。読めなければ手入力fallback
- **センタリング**（手持ち写真）: ラフな目安に留める。"PSA10判定"はしない（PSA含む業界AIでも管理光源下で苦戦）。人の肌感が主
- **スコープ肥大（DL011）**: MVPは①②に集中、③は補助線のみ。状態のML判定はPhase3以降

## 7. フェーズ
- **Phase0（今）**: 本設計を固める ← 今ここ
- **Phase1（7/8後）**: 写真→OCR型式→GR相場表示。核心価値
- **Phase2**: センタリング補助線
- **Phase3+（プロダクト化検討時）**: 状態ML・画像認識・ユーザー管理・法務/サポート
